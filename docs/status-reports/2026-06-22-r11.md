## Challenge findings (before building)

The kickoff asked to **keep the `buildServer` contract + error envelope identical, with no route
path/behavior change**, and named the 125 integration tests + 45 e2e as the regression net. I read
`server.ts`, `index.ts`, `test/helpers.ts`, and `cors.test.ts` and confirmed the path:

- **Riskiest assumption ‚Äî encapsulation transparency.** `app.register(plugin, ‚Ä¶)` puts a plugin's routes
  in a **child (encapsulated) context**. The one thing that could silently change behavior is whether the
  root's `setErrorHandler` (the `{ error: { message } }` envelope), the `@fastify/cors` hook, and the
  `addContentTypeParser` still apply inside those children. In Fastify 5 they **do** ‚Äî error handlers,
  hooks, and content-type parsers set on the root are **inherited** by child contexts unless a child
  overrides them, and the route plugins add **no** hooks/decorators/error-handlers of their own. So
  encapsulation is transparent here. The `cors.test.ts` preflight (handled by the root CORS hook before
  routing) and the per-area 404/409/400 assertions are exactly the tests that would catch a regression ‚Äî
  and they pass.
- **No `prefix` ‚Äî group logically, keep literal paths.** Domain areas deliberately cross URL roots:
  `/accounts/:id/credit-limit` and `/accounts/:id/original-principal` and `/envelopes/:id/target` are
  **analysis** reference-number setters, `/accounts/:accountId/transactions` is **transactions**,
  `/accounts/:accountId/reconciliations` is **reconcile**. Using a Fastify `prefix` per module would
  change paths. So modules group **by domain meaning** but every `app.<verb>("‚Ä¶")` keeps its **full
  literal path**. Zero path change.
- **Services constructed once.** `buildServer` builds a single `Services` object (one instance per
  service, as before) and passes it to every plugin via options ‚Äî no duplicate instantiation, identical
  DB wiring. Splitting routes did not multiply services.
- **Setters grouped with their analysis reports.** The existing file already placed credit-limit /
  original-principal / target in its analysis section (they are "the reference number for FEAT-014a /
  014b / 012"). `routes/analysis.ts` keeps that grouping, so `accounts.ts` / `envelopes.ts` stay pure
  CRUD and the FEAT-012/014 read+write pairs live together.

**Decision:** 9 per-domain `FastifyPluginAsync<{ services }>` modules + a `routes/shared.ts`, registered
from a 104-line `buildServer`; literal paths preserved; one service container shared by reference. Handlers
moved verbatim (byte-for-byte logic). No test added (refactor); the existing 125 API + 45 e2e are the net.

## Commands & gotchas (cold-start)

Gate: `npm run typecheck && npm run lint && npm test && npm run test:e2e && npm run format && npm run build --workspace @budgeteer/web && npm audit --omit=dev --audit-level=critical`

- **SCA note:** `npm audit` reports 3 **high**-severity advisories (kysely chain); the gate is
  `--audit-level=critical`, so it passes (exit 0). Pre-existing, not introduced by R11.
- **Encapsulation is transparent here, but it is real.** The route plugins are encapsulated children. The
  root error handler / CORS hook / content-type parser are **inherited** ‚Äî this works precisely *because*
  the plugins add no hooks/decorators of their own. If a future plugin needs a plugin-local hook or
  decorator, remember it will be scoped to that child (use `fastify-plugin` to break encapsulation if you
  need it global).
- **Add a new route by editing one file.** New `/accounts/*` route ‚Üí `routes/accounts.ts`; new analysis
  report ‚Üí `routes/analysis.ts`. Need a new service? Construct it once in `buildServer`'s `services` object
  and add it to the `Services` interface in `routes/shared.ts`.
- **Keep literal paths (no `prefix`).** Several setters live under `/accounts` or `/envelopes` but belong
  to `analysis.ts`. Don't "tidy" this with a Fastify `prefix` ‚Äî it would change the wire paths.
- `reuseExistingServer: !process.env.CI` ‚Äî kill stale `:3001`/`:5173` before e2e if you ran the preview.
