**Resume here:** `UX15` is **done, gate-green**, and it is the **final UX-Uplift item — the initiative
is complete**. The uplift was built desktop-first with graceful degradation already partly baked in
(the [cockpit](../features/cockpit.md) grid + Insights figure lists are `auto-fit minmax(...)` and
**already stack**; the shell nav + register rows already `flex-wrap`; the ADR-0007 chart SVGs are
`width:100%` and **already scale**), so this was a small **CSS/token-only** slice closing the
remaining phone-width gaps: **(1) reflow (WCAG 1.4.10)** — wide data tables no longer overflow the
page at ~320px but scroll within their own **keyboard-focusable** region, via a global **`.table-scroll`**
utility (`ui/base.css`, beside `.sr-only`) applied **once in `ChartFigure`** (every Insights table
flows through it as the chart's data-table fallback → one edit covers all nine) + the standalone
**envelope-ledger** table; each wrapper is `tabindex="0"` + named `role="group"` (axe
`scrollable-region-focusable`). `display:block` on `<table>` was **rejected** (strips AT table
semantics). **(2)** the **shell header** degrades to a narrow column (brand over the wrapping nav,
comfortable tap targets) and the **Insights sub-nav** — previously raw adjacent `<a>`s that touch when
wrapped — becomes a wrapping tab row with a ≥ 24px **target-size** (WCAG 2.5.8) per tab. **(3) page
padding** tightens ≤ 640px. Proven **axe-clean at 320px LIGHT AND DARK** with a wide table + the
reflow layout visible (a new "responsive reflow at phone width" e2e describe asserts **no horizontal
page scroll** + the focusable region). **No ADR/spike/dep**, no data/API/domain/chart change. Gate
green: **366 Vitest passing** (+2; 1 pre-existing unrelated carry — §4) **+ 94 e2e** (+2), build
**117.27 KB gz** (+0.07 KB, **no dependency**; < 120 KB, ~2.7 KB headroom). **Next:** the UX Uplift is
**complete** — no remaining Uplift items; see §5 for post-Uplift candidates (owner's call).

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **ADR-or-not / spike-or-not** | **No ADR, no spike, no new dependency.** Media queries (`max-width: 640px`) + one global CSS utility on the existing UX3/UX4/UX8 modules. §11 fast-path; the FEAT note is the paperwork. |
| **"Nav collapses" — scope call (flagged)** | The roadmap line says "nav collapses." A true hamburger adds a JS disclosure + focus management — a **new interactive a11y surface**, at odds with this slice's *COMPRESS / low-risk / desktop-first-not-mobile-first* mandate (initiative §1). The shell nav **already wraps**; this slice makes that wrap tidy (stacked column, tap targets) = graceful degradation. A hamburger remains an available follow-on if the owner wants it. |
| **Reflow the right way** | Data tables are the standard WCAG 1.4.10 reflow *exception* (may scroll). The fix scopes scroll to the table's own region so the **page** never scrolls, and keeps that region keyboard-operable. `display:block` on `<table>` was rejected — it can strip native table semantics in some AT; a wrapper preserves them. |
| **Reality before paper** | The **narrow (320px) axe scan caught** the Insights sub-nav's target-size gap (raw adjacent links that touch when wrapped) — invisible at desktop width. Styled + fixed in the same change. This is exactly why the slice's own point (narrow-viewport a11y) became a test. |
