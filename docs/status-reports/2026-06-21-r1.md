name; clicking "Save" calls `PATCH /accounts/:id` and refreshes the list. Mirrors the template
rename pattern. **Gate: 227 Vitest + 24 e2e** (was 227 + 23).

## What R1 built

**No API work.** `PATCH /accounts/:id` was already implemented and tested (service + route + 409
handling). The gap was purely in the client layer.

**Three-part delivery:**

1. **`api.ts`** — added `renameAccount(id: string, name: string): Promise<AccountView>` to the `Api`
   interface + `httpApi` implementation (`PATCH /accounts/${id}` with `{ name }` body, response
   unwrapped from `{ account: AccountView }`).

2. **`test/fakeApi.ts`** — added `renameAccount(id, name)`: finds the account by id, mutates its
   name in-place, returns a copy. Consistent with the other mutation methods.

3. **`Dashboard.tsx`**:
   - Added `refreshAccounts()` (mirrors `refreshEnvelopes()`) and `renameAccount(id, name)` handler
     (calls `api.renameAccount` then `refreshAccounts`; surfaces errors via `setLoadError`).
   - `AccountList` gains `onRename?: (id: string, name: string) => void` prop + internal
     `renamingId`/`renameValue` state. Each account row shows a "Rename" button
     (`aria-label="Rename {name}"` for a11y); clicking it opens an inline input (pre-filled with
     the current name); clicking "Save" calls `onRename` and clears `renamingId`.
   - `Dashboard` passes `onRename={(id, name) => void renameAccount(id, name)}` to `AccountList`.

4. **`e2e/accounts.spec.ts`** — added `rename an account inline (R1)` test: create account →
   click Rename button → fill new name → click Save → assert new name button visible + new Rename
   button visible.

5. **`e2e/setup.ts` + all e2e specs** — the Rename button's `aria-label="Rename {name}"` caused
   Playwright's partial-name matching to resolve two elements for every `getByRole("button", { name:
   accountName })` call (the name button AND the rename button). Fixed by adding `exact: true` to:
   - `setup.ts` `createAccount` assertion (line 16)
   - 13 account-button lookups across `accounts.spec.ts`, `transactions.spec.ts`,
     `transfers.spec.ts`, `reconcile.spec.ts`, `templates.spec.ts`, `recurring.spec.ts`,
     `analysis.spec.ts`, `envelopes.spec.ts`
