**Resume here:** Account/envelope management is **demoted off the home**. The home (`/`) now renders
the **UX5 cockpit only** (`Home.tsx`). The `/accounts` and `/envelopes` **LIST routes** (deferred
from UX3, built now) own per-entity CRUD — a **progressive "Add" affordance**, each name a `<Link>`
to its detail (UX3 left these as buttons), inline rename/archive (+ the R5 inline budget on
envelopes). The new `/manage` **hub** owns the cross-entity tools — the **R4 net-worth summary**
(spans all accounts) and **Move-money** (between two envelopes) — and links to the two list pages.
The shell nav gains **Accounts · Envelopes · Manage**. Everything **fans out over existing reads — no
new endpoint, no schema/API/domain change**; figures reconcile to the ledger. Gate green: **304
Vitest + 61 e2e** (incl. axe light + dark on all three new surfaces), build 93.4 KB gz (< 120 KB).
**Next: `UX7`** — global quick-add transaction (always-available entry, modal route).

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Surface-split fork (resolved)** | Owner chose **list routes own per-entity CRUD; `/manage` is the cross-cutting hub** over everything-on-`/manage`. Reads the brief's "and" as a clean, non-duplicated distribution: net-worth (Σ all accounts) and Move-money (between envelopes) are the only genuinely cross-entity tools, so they sit on `/manage`; everything tied to one account/envelope lives with its list. No list is rendered twice. |
| **`link-in-text-block` (fixed)** | The `/manage` hub first listed each link followed by descriptive prose in the same `<li>` → axe WCAG 1.4.1 flagged the link as not distinguished from surrounding text (light AND dark). Moved the prose to a standalone intro `<p>`; each hub link now stands alone. The account/envelope list-item links pass because their siblings are elements (kind/balance/buttons), not prose. |
| **Carries retired** | UX5 flagged two carries that UX6 resolves: the redundant net-worth (home cockpit panel **and** the R4 summary) — the R4 summary now lives only on `/manage`; and the within-mount freshness gap — management is off the home, and each route remounts (refetches) on navigation. |
| **Shared dev store** | `PGLITE_DIR` in `.env` persists a noisy dev store; the live screenshots showed real-but-noisy figures (negative aggregate assets from accreted e2e data). The e2e run uses a throwaway store (`playwright.config.ts`) and asserts invariants, so it is robust. Do **not** `db:reset` to "tidy" — it deletes dev data. |

## 6. Commands & gotchas (cold-start)

```sh
npm install
npm run typecheck && npm run lint && npm run format
npm test                 # 304 Vitest (node + jsdom)
npm run test:e2e         # 61 Playwright (incl. e2e/a11y.spec.ts light+dark on the new surfaces)
npm run build --workspace @budgeteer/web   # 93.4 KB gz JS, 2.45 KB gz CSS
```
- **Surfaces:** `/` = `Home.tsx` (cockpit only) · `/accounts` = `AccountsList.tsx` · `/envelopes` =
  `EnvelopesList.tsx` · `/manage` = `ManageView.tsx`. `Dashboard.tsx` is gone.
- **e2e navigation:** use the `setup.ts` helpers — `createAccount`/`createEnvelope` (drive the
  progressive Add on the list routes), `openAccount`/`openEnvelope` (click the name `<Link>`),
  `goToAccounts`/`goToEnvelopes`/`goToManage`/`goToDashboard`. List-item names are **links**, not
  buttons. Move-money + net-worth are on `/manage`; envelope balances on `/envelopes`.
- **Progressive Add:** the form is hidden behind an "Add account"/"Add envelope" button; the helper
  reveals it if `form.isVisible()` is false, then submits the scoped form button (same label).
- **a11y:** keep hub/intro links out of prose (`link-in-text-block`); never animate opacity on a text
  wrapper (the shell's transform slide is the only motion); one banner + one `<h1>`/`<main>` per route.
- **Fan-out, not an aggregate:** no `home-summary`/`manage-summary` endpoint — compose existing reads.
- **Perf test flakiness:** `apps/api/test/perf.test.ts` (`GET /accounts` p95 < 50 ms) can exceed the
  budget under heavy concurrent load; it passes in isolation (p95 ≈ 1 ms). Re-run if it trips.
