## 6. Commands & gotchas (cold-start)

```sh
npm install
npm run typecheck && npm run lint && npm run format
npm test                 # 296 Vitest (node + jsdom)
npm run test:e2e         # 55 Playwright (incl. e2e/a11y.spec.ts light+dark, e2e/cockpit.spec.ts)
npm run build --workspace @budgeteer/web   # 93.0 KB gz JS, 2.45 KB gz CSS
```
- The cockpit lives in `apps/web/src/Cockpit.tsx`; it is rendered by `Dashboard.tsx` (so the home
  keeps one `<main>`/`<h1>`). It takes `api` as a prop (unit-tested directly) and uses `<Link>` →
  **its tests, and `Dashboard`/`MoveMoneyForm` tests, wrap renders in `MemoryRouter`.**
- **Fan-out, not an aggregate:** the cockpit composes existing endpoints. Don't add a `home-summary`
  endpoint unless a perf budget (`07_NFR`) demands it — the brief's open Q resolved to fan-out.
- **Insights view slugs have no hyphen:** net worth is `/insights/networth` (not `net-worth`); the
  others are `spend`/`budget`/`forecast`/`credit`/`payoff` (`AnalysisSection.tsx` `TABS`).
- **Budget panel reconciliation:** sum over **targeted** envelopes only (`getBudgetVsActual`'s
  `totalSpent` also counts untargeted spend → would break budgeted − spent = remaining).
- **e2e link names:** the cockpit's "Manage recurring" substring-matches "Recurring" → `openRecurring`
  uses `exact: true`. Keep new cockpit link labels distinct from the shell nav labels.
- **Shared dev store:** `PGLITE_DIR` is set in `.env`, so `npm run dev`/e2e share one persistent
  PGlite (and the e2e tests reuse a running server). Manual screenshots show real but noisy data;
  don't `db:reset` to "clean up" — it deletes the user's dev data. The e2e assert invariants
  (net = assets + liabilities), so they're robust to the shared store.
