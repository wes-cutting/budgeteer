**Resume here:** The **Insights phase is unblocked.** `UX2` ran as a **throwaway spike** (findings, not
production code — the V1 app is untouched) to answer the one question that gates `UX8`: *can a chart be
fully accessible, with a data-table fallback, in light AND dark, within the 120 KB gz bundle?* Built **one
real chart** (net-worth-over-time, the `getNetWorth` shape) **hand-rolled in SVG** with the production
design tokens, then measured it in **real Chromium**. **Answer: yes, hand-rolled — 0 axe violations**
(WCAG 2.2 AA) across **light/dark × table-shown/hidden**, keyboard- and screen-reader-reachable via a real
`<table>` fallback, with **color never the sole signal** (line color + dash pattern + marker shape + direct
label). **The decider was bundle:** hand-rolled adds **1.94 KB gz**; **Recharts alone is 129 KB gz** —
larger than the entire budget — so a charting library is **disqualified**. [`ADR-0007`](../adr/ADR-0007-accessible-charting.md)
(hand-rolled SVG + img-summary/table-fallback contract + new `--chart-*` tokens + a shared `Chart` primitive)
→ **`Validated`**. App gate still green: **311 Vitest + 66 e2e**, build **105.4 KB gz**. **Next: `UX8`** —
Analysis → Insights migration (presentation-only; the six reads/endpoints are done), now **`Ready`** on the
ADR-0007 contract.

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Library vs hand-rolled — hand-rolled (resolved)** | The 120 KB budget with ~14.6 KB headroom forces it: Recharts measured **129 KB gz** for a basic multi-line chart (React-externalized) — larger than the whole app budget; even lean libs (uPlot ~40 KB, visx modular) eat most/all of the headroom for *one* of six charts. Hand-rolled adds **1.94 KB gz** and gives total a11y control. Same restraint rationale as ADR-0005 (tokens/CSS-Modules over Tailwind). |
| **The a11y pattern — "chart as `img` + table fallback"** | SVG `role="img"` + concise summary `aria-label`; decorative innards `aria-hidden`; a native `<table>` is the keyboard/SR source of truth (no ARIA grid). This is the GOV.UK/Highcharts-style recommended pattern and is **additive over the existing analysis grids** — UX8 keeps the table, adds an SVG above it. |
| **Color never the sole signal** | Each series is distinguished four ways (color + dash + marker shape + direct label), so WCAG 1.4.1 holds even in grayscale; `--chart-*` stroke tokens are *also* ≥ 3:1 (1.4.11) in both schemes, but a11y does not depend on color discrimination. |
| **Residual (low)** | Validated on a line/time-series. Bars (spend-by-envelope, budget-vs-actual) and gauges/progress (credit, payoff) reuse the shape-agnostic contract; UX8 builds each shape on it and re-runs the axe gate (light+dark) per shape. No follow-up spike. |
| **Throwaway** | The harness is disposable (findings persist in SPIKE-07 + ADR-0007). Discard `spikes/07-accessible-charting/` once UX8 absorbs the pattern (open follow-up). |

- **The pattern UX8 must follow** (ADR-0007): SVG `role="img"` + summary `aria-label`, `aria-hidden`
  innards; a real `<table>` fallback (keyboard/SR source of truth); color reinforced by dash/shape/label;
  `--chart-*` tokens (≥ 3:1) with a dark variant; **no opacity animation on text**; honor reduced-motion.
- **Bundle:** budget 120 KB gz, currently 105.4 KB (~14.6 KB headroom). The hand-rolled `Chart` primitive
  is ~1.94 KB gz total (not per chart). **Do not reach for a charting library** — it breaks the budget.
- **a11y gate is light AND dark** — `e2e/a11y.spec.ts` already scans both; add each new Insights chart to it.
- **Shared dev store (carry):** `PGLITE_DIR` persists a noisy dev store; e2e uses its own throwaway store.
  Do **not** `db:reset` to "tidy" — it deletes dev data.
- **Perf test flakiness (carry):** `apps/api/test/perf.test.ts` can trip under heavy concurrent load;
  re-run if it trips (passes in isolation).
