**Resume here:** The **Insights phase has its first NEW visualisation.** `UX9` is a **vertical
slice** that adds a "**where did the money go?**" view at **`/insights/breakdown`** (a 7th sub-nav
tab, "Breakdown", next to Spend): for a chosen month, each envelope's **share of total OUTFLOW** as
**ranked horizontal bars**, so proportion reads at a glance. It's built on the UX8 **`ui/Chart`**
primitive with a **new `BreakdownBars` shape** (through the same `ChartFigure` a11y scaffold).
**Read decision: it composes the EXISTING `getBudgetVsActual(month)` read — NO new endpoint /
schema / API / domain change** (R4/R5/UX5 fan-out precedent). That read's `spentCents` is **pure
outflow** (withdrawal allocations only; funding excluded; refunds netted) — the correct breakdown
numerator, unlike spend-by-envelope's *net* flow which would conflate funding with spend. **The
hardest a11y case (a many-category breakdown, no colour-only) is solved by giving colour NO
categorical meaning at all**: every bar is one colour (`--chart-1`), and each category reads from a
**direct text label** (rank · name · outflow · share) + **rank order** + bar length. `role="img"` +
one-line summary, `aria-hidden` innards, the ranked **table** (envelope · outflow · % share) as the
SR/keyboard source of truth; shares are taken over the displayed positive slices so they **sum to
100%**. `e2e/a11y.spec.ts` now **seeds real outflow** and is **axe-green LIGHT AND DARK** with the
chart rendered. Gate green: **320 Vitest + 75 e2e**, build **108.93 KB gz** (< 120 KB). **Next:
`UX10`** — new viz: spending trends over time, `Planned` on the `ui/Chart` primitive.

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Read decision (the headline)** | Composed **`getBudgetVsActual`**, *not* the kickoff-suggested `getEnvelopeSpend`. Both are existing reads (no endpoint either way), but `getBudgetVsActual.spentCents` is **pure outflow** (funding excluded, refunds netted) — the correct numerator. `getEnvelopeSpend` returns *net* flow, whose "negative component" understates gross outflow whenever an envelope was funded **and** spent in the same period. Same fan-out spirit, semantically-correct source. |
| **Many-category a11y** | The hard case is "no colour-only" with N envelopes — there aren't N accessible, ≥ 3:1, pattern-friendly hues. Resolved by making colour **decorative only**: one bar colour, and every category distinguished by a **direct text label** + **rank order** + **bar length**. A new *shape*, not a new contract → **no ADR change** (ADR-0007 is shape-agnostic). A pie/donut was rejected (labelling + no-colour-only are far harder). |
| **Share denominator** | Shares use Σ of the **displayed positive-outflow slices**, not the report's `totalSpentCents`, so percentages sum to 100% even if a refund nets a row ≤ 0 and it drops out. Truthful and self-consistent. |
| **Cockpit** | Left **unchanged** — the cockpit's budget panel already deep-links into Insights, and a breakdown panel doesn't fit a slot without scope creep; the sub-nav surfaces the view. (Kickoff: "if it fits".) |
| **e2e robustness** | The shared e2e store accretes current-month outflow across parallel specs, so the household-wide total (hence exact shares) isn't deterministic. The functional test asserts **robust facts** — each envelope's own outflow + that the bigger spender ranks above the smaller — mirroring the net-worth test's invariant approach. |
| **Verification** | Chart is rendered-and-axe-scanned in **real Chromium** by the e2e suite (seeded with real withdrawals). The interactive preview server was again unusable (another session holds the dev port); the Playwright suite is the authoritative real-browser verification, as in UX8. |
| **Perf test (carry)** | `apps/api/test/perf.test.ts` can trip under heavy concurrent load in the full run; passes in isolation. Re-run if it trips — not a regression. |
| **Shared dev store (carry)** | `PGLITE_DIR` persists a noisy dev store; e2e uses its own throwaway store. Do **not** `db:reset` to "tidy". |

  summary `aria-label`, `aria-hidden` innards; a real `<table>` is the keyboard/SR fallback (passed
