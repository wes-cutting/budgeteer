## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Validation depth** | `HOST` is a trimmed non-empty string, not an IP/hostname grammar — Fastify/Node reject an unresolvable bind address loudly at `listen()` anyway, so a stricter schema would only duplicate that failure with a worse message for exotic-but-valid values (IPv6, hostnames) |
| **Why SECURITY.md §3** | The bullet lives under *Authentication & authorization*, not *Operational*: the review's framing is that the reachable surface must scale with the auth story. This narrows the 2026-06-15 "anyone reaching `:3001`" acceptance — that acceptance now describes loopback, not the Wi-Fi |
| **`#19` unchanged** | The auth/household-scoping epic stays `Deferred`; opting into `0.0.0.0` is now its documented pull-forward trigger (recorded in SECURITY.md, `.env.example`, and the roadmap row) |
| **e2e flake (watch, not a carry)** | First full run: `envelopes.spec.ts:12` timed out clicking "Add envelope" — Playwright's log shows `element is not stable` (layout shift under parallel load); the page itself was fully rendered.
