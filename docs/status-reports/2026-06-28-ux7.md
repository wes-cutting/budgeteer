**Resume here:** PRD **journey #1 (the common case) is no longer buried** behind opening an account
register. The persistent shell nav now has an always-available **Add transaction** action that opens a
**modal route** at `/transactions/new` (a child of the shell): pick the **account**, enter
amount/payee, allocate in **Single or Split**, save, and you're **returned to where you were**. It
**reuses the register's editor verbatim** — `AddTransactionForm` gained an optional `accounts` picker
mode and `AllocationEditor` an optional `disabled` gate (both default to the unchanged register
behavior) — and **fans out to the existing `createTransaction`** (no new endpoint, no schema/API/domain
change); a **partial** allocation's remainder still surfaces in needs-allocation. The modal is the new
**`Dialog` primitive** (`apps/web/src/ui/Dialog.tsx`, `@radix-ui/react-dialog` per ADR-0005) — the
first genuine modal, so the deferred UX4 `Dialog` of seed-and-grow lands here; Radix supplies the
focus trap / ESC / overlay-close / focus-restore / `role="dialog"`, **axe-clean light AND dark**. Gate
green: **311 Vitest + 66 e2e**, build **105.4 KB gz** JS (< 120 KB; +~12 KB from Radix dialog).
**Next: `UX8`** — Analysis → Insights migration, **gated first by `UX2`** (the accessible-charting
spike → `ADR-0007`).

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **`Dialog` build — Radix (resolved)** | ADR-0005 (Validated) reserves **Radix for the hard a11y widgets and names Dialog**, and SPIKE-06 validated the Radix dialog axe-clean. So `@radix-ui/react-dialog` is the **doc-aligned, lowest-a11y-risk** path (focus trap / ESC / overlay-close / focus-restore / `role="dialog"` for free). Cost: +~12 KB gz → 105.4 KB total, well under the 120 KB budget ADR-0005 measured against. Hand-rolling would have diverged from the ADR and raised a11y risk for no real saving. |
| **Modal mechanism — child route (resolved)** | With a data router (`createBrowserRouter`), a true `backgroundLocation` overlay is awkward. A **child route under the shell + a portal'd `Dialog` + history-safe back** delivers the full modal contract (trap / ESC / return-focus / "return to where you were") cleanly. The Radix **portal to `document.body`** escapes the keyed `.content` transform, so there is no `position:fixed`-inside-`transform` glitch. |
| **Entry point — shell nav only (resolved)** | The brief allows "shell nav **and/or** the cockpit's Needs-allocation panel". Shell-nav-only chosen: it is always-available and **return-focus-solid** (the trigger stays mounted across the route change), and it avoids adding a link inside the cockpit `NeedsPanel` body that would break its "all-clear → no link" reconciliation test for no real gain. |
| **`aria-modal` note** | Radix v1.1.17 wires modality via `aria-labelledby` + a trapped focus scope + background-inerting rather than emitting an `aria-modal` attribute. No axe rule requires `aria-modal`; the dialog is correctly named and modal in behavior (verified by the light+dark axe scans). |
| **Shared dev store** | Unchanged from UX6: `PGLITE_DIR` in `.env` persists a noisy dev store; the live preview used a **throwaway** PGlite dir. The e2e run uses its own throwaway store and unique fixture names. Do **not** `db:reset` to "tidy" — it deletes dev data. |

## 6. Commands & gotchas (cold-start)

```sh
npm install
npm run typecheck && npm run lint && npm run format
npm test                 # 311 Vitest (node + jsdom)
npm run test:e2e         # 66 Playwright (incl. e2e/a11y.spec.ts open-modal light+dark)
npm run build --workspace @budgeteer/web   # 105.4 KB gz JS, 2.70 KB gz CSS
```
- **Quick-add:** `Add transaction` (shell nav) → `/transactions/new` = `QuickAddTransaction.tsx`, a
  child route rendering the `Dialog` (`apps/web/src/ui/Dialog.tsx`, Radix) + the reused
  `AddTransactionForm` in **account-picker mode** (`accounts` prop) → `AllocationEditor`.
- **Reuse seams:** `AddTransactionForm` `accounts?` enables the picker + gates Save; `AllocationEditor`
  `disabled?` is the external Save gate. **Both default to the unchanged register/needs behavior** —
  the register passes neither.
- **e2e:** `openQuickAdd(page)` opens the modal from the nav; the dialog is portaled to body, so query
  it by `getByRole("dialog", { name: "Add a transaction" })`. The a11y scan presses `Escape` at the
  end so the shared `afterEach` (nav back to home) isn't blocked by the open modal.
- **a11y:** never animate opacity on a text wrapper; the Radix portal escapes the shell transform.
  Radix doesn't emit `aria-modal` in this version — assert the **accessible name** (`aria-labelledby`),
  not `aria-modal`.
- **Bundle:** Radix dialog added ~12 KB gz (now 105.4 KB; budget 120 KB in `07_NFR`). Watch the budget
  as more Radix primitives (Menu/Toast/Tabs) land in later slices.
- **Preview gotcha:** the web must be served on **:5173** (the API CORS allowlist origin). For a manual
  preview, free :5173 and set `autoPort: false` on the `web` launch config (`.claude/` is gitignored).
- **Perf test flakiness (carry):** `apps/api/test/perf.test.ts` (`GET /accounts` p95 < 50 ms) can trip
  under heavy concurrent load; it passes in isolation (p95 ≈ 1 ms). Re-run if it trips.
