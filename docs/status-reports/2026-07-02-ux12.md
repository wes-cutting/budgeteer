**Resume here:** The **first slice of the UX Uplift Polish track** is done. `UX12` as written bundles
**four independent polish threads** (skeletons · toasts · destructive-action confirms · inline
validation); per *vertical, not horizontal* (§5) and right-size/compress (§11) this slice ships
**one** — **confirmation on destructive actions** — and leaves the other three tracked (none dropped).
The three previously **one-click, no-confirm** destructive actions now open a **confirm dialog** first:
**Archive envelope** (`/envelopes`), **Archive account** (`/accounts`), and **Delete template**
(`/templates`, the only *irreversible* one). Owner decided up front (this session): **all three get a
consistent blocking confirm** (not just the irreversible Delete). The only new code is one primitive,
**`ConfirmDialog`** (`apps/web/src/ui/ConfirmDialog.tsx`) — a **controlled/transient** modal built on
the **same SPIKE-06-validated Radix `Dialog`** the quick-add uses (reuses `Dialog.module.css` + a
`.actions` row; **no new dependency**, **+0.42 KB gz**). It differs from the route-driven `Dialog` only
in lifecycle: `open` is a prop toggled by a row click, and it closes on confirm/cancel. **State never
rests on colour:** the `danger` button variant is emphasis only — the action word ("Archive"/"Delete")
**and** the dialog title carry the meaning; **Cancel is first in DOM** so it's the safe default focus;
**ESC/overlay = cancel**, the mutation runs only on the explicit confirm. `e2e/a11y.spec.ts` scans the
confirm dialog **open, LIGHT AND DARK**. **No ADR/spike** (reuses a validated primitive, no dep, no
data/API/domain/chart change — §11 fast-path). Gate green: **340 Vitest passing** (+5; 1 pre-existing
unrelated carry — see §4) **+ 83 e2e** (+2), build **111.48 KB gz** (< 120 KB). **Next:** finish the
remaining `UX12` threads (skeletons `UX12b` · toasts `UX12c` · inline validation `UX12d`), then
`UX13`–`UX15`.

- **`ConfirmDialog` is the reusable destructive-confirm primitive** (`apps/web/src/ui/ConfirmDialog.tsx`):
  controlled `open`, `onConfirm`/`onCancel`, `danger` confirm button. Reuse it for any future
  irreversible/disruptive action; **don't** reach for the route-driven `Dialog` (that's for modal
  *routes*). State must never lean on the `danger` colour — keep the action word in the label + title.
- **The two Dialog flavours:** `Dialog` = route-driven modal (hard-`open`, dismissal navigates);
  `ConfirmDialog` = controlled/transient (open on demand, close in place). Same Radix base + CSS.
- **a11y gate is light AND dark** — to scan a transient dialog, open it, `expect(...dialog...).toBeVisible()`,
  `assertNoViolations(page)`, then **ESC to close** so the shared afterEach nav isn't blocked (the
  quick-add scan is the pattern; the UX12 confirm scan mirrors it).
- **Selector hygiene:** the dialog's confirm button is labelled just **"Archive"/"Delete"** while the
  row trigger is **"Archive {name}"** — scope dialog clicks to `getByRole("dialog", { name: … })` and
  use `exact: true` in Playwright to avoid the substring match.
- **Bundle:** 120 KB gz ceiling, 111.48 KB now (~8.5 KB headroom). `ConfirmDialog` added **no
  dependency** (reuses the Radix Dialog UX7 already bundled).
- **Known carries (not this slice's regressions):** `recurring.test.ts`'s post-due occurrence-count
  case, and a flaky `getByLabel("Account")` collision in the forecast e2e — both reproduce on `main`,
  see §4.
