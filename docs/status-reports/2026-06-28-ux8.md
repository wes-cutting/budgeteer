**Resume here:** The **Insights phase has its charts.** `UX8` was a **vertical, presentation-only slice**:
the six analysis views are **renamed** ("Insights — …"), **restyled** onto the UX4 design system, and each now
renders a **hand-rolled accessible chart** above its existing table — on the ADR-0007 contract (hand-rolled SVG +
a data-table fallback, **no charting library**). A new shared **`apps/web/src/ui/Chart.tsx`** owns the a11y
contract once (`ChartFigure` + `LineChart`/`BarChart`/`Gauge`); new **`--chart-1/2/3`/`--chart-grid`** tokens give
≥ 3:1 strokes in light + dark. Net worth + cash-flow = **line**, spend + budget = **bar**, credit + payoff =
**gauge**. Every chart is `role="img"` with a one-line summary, `aria-hidden` innards, the **existing table as the
keyboard/SR fallback**, and **colour is never the sole signal** (dash / marker / hatch / bar-direction / direct
labels). `e2e/a11y.spec.ts` now **seeds data per shape** and is **axe-green LIGHT AND DARK with every chart
rendered** (the old insights a11y tests scanned empty states only). **No new endpoint / schema / API / domain
change.** Gate green: **315 Vitest + 72 e2e**, build **108.33 KB gz** (< 120 KB). `ADR-0007` → **`Accepted`**; the
SPIKE-07 harness was discarded. **Next: `UX9`** — new viz: spending breakdown (per-envelope share of outflow for a
period), `Planned` on the `ui/Chart` primitive.

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Chart shape per view** | Credit/payoff render **one aggregate gauge** (overall utilization / payoff, with a 100% threshold marker); the per-account figures + month trends stay in their tables (detail + fallback) — keeps the slice tight vs. many tiny gauges. | **Table-as-slot integration** | The `Chart` primitive takes each view's **existing table as a slot** (one table, inline editors preserved), faithful to ADR-0007's "add an SVG above the table". The disclosure toggle defaults **shown** so the budget/credit/payoff editors stay reachable. |
| **a11y gate now actually covers charts** | The previous insights a11y tests navigated with **no data** → they scanned empty states, never the charts. UX8's seeded scans close that gap (and add dark). |
| **Verification** | Charts are rendered-and-axe-scanned in **real Chromium** by the e2e suite (seeded per shape).
