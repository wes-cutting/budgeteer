**Resume here:** **`R15` is `Done` and gate-green** — clicking an envelope name on the Dashboard
opens a newest-first allocation list that answers "what transactions are in this envelope and why
is its balance what it is?" `GET /envelopes/:id/ledger` joins `allocations ⋈ transactions ⋈ accounts`,
scoped to the household; each row carries date · payee/memo · source account · signed amount
(`+` = funded, `−` = spent). No schema change. **Gate: 227 Vitest + 23 e2e** (was 218 + 22).

## What R15 built

**No spike — no unproven assumption.** The query is a read-only join over three existing tables;
the only design decision (envelope-transfer reallocations out of scope for V1) was already settled
in the spec.

**Four-part delivery:**

1. **`envelopeService.ts`** — added `ledger(id)` method + exported `EnvelopeLedgerRow`:
   - Checks envelope exists (household-scoped); throws `NotFoundError` → 404 if missing.
   - Joins `allocations as a` → `transactions as t` → `accounts as ac`; filters by
     `a.envelope_id = id` and `t.household_id = DEFAULT_HOUSEHOLD_ID`.
   - Orders `t.occurred_on DESC, t.created_at DESC` (newest first, stable on same-date tie).
   - Converts `amount_cents` (IntCents string) via `Number()`; `occurred_on` via `toDateStr()`
     (handles both `Date` from node-postgres and string from PGlite).
   - `transaction_kind` cast to `"opening"|"normal"|"transfer"` — same pattern as `transactionService`.

2. **`GET /envelopes/:id/ledger` in `server.ts`**:
   - Returns `{ rows: EnvelopeLedgerRow[] }`; `404` on `NotFoundError`.
   - Placed before the archive/unarchive routes (Fastify parses `/ledger` before `/:id/archive`
     since literal segments beat wildcards in the same path depth).

3. **Web (`apps/web/src/`):**
   - `api.ts`: added `EnvelopeLedgerRow` type + `getEnvelopeLedger(envelopeId)` to `Api` interface
     + `httpApi` impl (`GET /envelopes/${envelopeId}/ledger`).
   - `EnvelopeLedger.tsx`: new component mirroring `AccountRegister`'s structural pattern —
     `useEffect` load on `envelope.id`, loading/empty/error/populated states, `<table>` with
     `<th scope="col">` headers, `aria-labelledby`, `role="alert"` on errors, "Try again" retry,
     "(archived)" badge in heading, `+`/`−` prefix on amounts (text, not color-only). `← Dashboard`
     button calls `onBack`.
   - `Dashboard.tsx`: added `onOpenEnvelope?: (envelope: EnvelopeView) => void` prop; passed it
     to the `EnvelopeList` sub-component; envelope names in both active and archived lists are now
     `<button>` elements when `onOpen` is provided (mirrors the `AccountList` pattern).
   - `App.tsx`: added `{ name: "envelope"; envelope: EnvelopeView }` to the `View` union; renders
     `<EnvelopeLedger>` when active; passes `onOpenEnvelope` to `<Dashboard>`.
   - `test/fakeApi.ts`: added `getEnvelopeLedger` — iterates `txns` sorted newest-first, collects
     matching allocations for the requested envelope id.

4. **Tests:**
   - `apps/api/test/envelope-ledger.test.ts` (5 integration tests): empty envelope → 200 + `[]` ·
     newest-first order · deposit positive / withdrawal negative · `opening`-kind rows included ·
     split across two envelopes (each ledger scoped correctly) · missing envelope → 404.
   - `apps/web/src/EnvelopeLedger.test.tsx` (4 component tests): renders rows with date/payee/account/amount ·
     empty state shows message · archived badge appears · back button calls `onBack`.
   - `e2e/envelopes.spec.ts` extended (+1 e2e): create account + envelope → enter withdrawal →
     click envelope name on Dashboard → see ledger row (Whole Foods / account name visible) →
     back button returns to Dashboard.

## Commands & gotchas (cold-start)

Gate is unchanged: `npm run typecheck`, `npm run lint`, `npm test`, `npm run test:e2e`,
`npm run format`, `npm run build --workspace @budgeteer/web`.

- The `GET /envelopes/:id/ledger` route must be registered **before** the archive/unarchive routes
  in `server.ts` to prevent Fastify treating `"ledger"` as an `:id` param value. (Literal segments
  take priority over named params in the same position, so order doesn't actually matter in Fastify —
  but placing it nearby for readability is good practice.)
- Envelope-transfer reallocations (`envelope_transfers`) are intentionally **excluded** from the
  ledger in V1. The balance header accounts for them; the ledger shows only `allocations` rows.
  If a user wonders why the balance differs from the sum of rows, this is why.
- The `(archived)` badge is appended to the heading text, not a separate element — this is intentional
  so it reads naturally in both visual and screen-reader contexts without needing ARIA extras.
