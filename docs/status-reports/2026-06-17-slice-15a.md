## What #15a built

**No spike ‚Äî no unproven assumption.** Export is a parallel SELECT across 15 tables; the
only care is serializing BigInt-string cents as `Number()` (exact to ~$90T per EH6/ADR-0003).

**Four-part delivery:**

1. **`apps/api/src/services/backupService.ts`** ‚Äî `makeBackupService(db)`:
   - Parallel `Promise.all` over 15 table queries (households, accounts, envelopes,
     transfers, envelope_transfers, transactions, allocations, templates, template_lines,
     recurring_transactions, recurring_lines, reconciliations, envelope_targets,
     credit_limits, loan_principals).
   - The three child tables without `household_id` (allocations, template_lines,
     recurring_lines) are scoped via an inner join to their parent table.
   - Views (`v_account_balances`, `v_envelope_balances`) are excluded ‚Äî they're derived.
   - `normRow()` converts Kysely's BigInt-string cents columns to `number` (via `Number()`);
     Date objects serialize to ISO strings natively via `JSON.stringify`.
   - Returns a typed `BudgeteerBackup` (`version: 1`, `exportedAt`, `householdId`, `tables`).

2. **`GET /export` in `apps/api/src/http/server.ts`**:
   - Sets `Content-Disposition: attachment; filename="budgeteer-backup-YYYY-MM-DD.json"`.
   - Sets `Content-Type: application/json` explicitly (via `reply.type()`).
   - Sends `JSON.stringify(snapshot, null, 2)` ‚Äî pretty-printed; Fastify sends the string
     as-is without re-serializing.
   - The response body is **never logged** (financial data). Error path ‚Üí 500 (caught by
     Fastify's global error handler).

3. **Web (`apps/web/src/`):**
   - `api.ts`: exports `exportUrl = \`${BASE}/export\`` (a string constant, not a fetch method).
   - `Dashboard.tsx`: imports `exportUrl`; adds `<a href={exportUrl}>Download backup</a>` in
     the header alongside the existing navigation buttons. Plain anchor ‚Üí browser navigates to
     the URL ‚Üí `Content-Disposition: attachment` triggers the download; the current page stays.
     No fetch, no CORS preflight, no JavaScript download logic needed.

4. **Tests:**
   - `apps/api/test/backup.test.ts` (4 integration tests): 200 + Content-Disposition regex ¬∑
     cents are numbers not strings ¬∑ empty DB returns household row + empty arrays ¬∑ seeded
     accounts/envelopes/allocations appear in the snapshot.
   - `e2e/backup.spec.ts` (2 e2e tests): "Download backup" link visible on the dashboard ¬∑
     `GET /export` returns 200, correct Content-Disposition, valid JSON with `version: 1` and
     a non-empty `tables.accounts`.

## Commands & gotchas (cold-start)

Gate is unchanged: `npm run typecheck`, `npm run lint`, `npm test`, `npm run test:e2e`,
`npm run format`, `npm run build --workspace @budgeteer/web`.

- The API server (`npm run start`) must be restarted after API changes (no watch mode; R10
  adds it). The `GET /export` route is only available once the server is running.
- The web `<a href={exportUrl}>` uses `http://localhost:3001` by default (from `VITE_API_BASE_URL`
  or the fallback in `api.ts`). If the API port changes, set `VITE_API_BASE_URL` in `.env`.
- `normRow()` converts only the known cents columns (a hard-coded Set). If a future migration
  adds a new `*_cents` column, add its name to `CENTS_COLUMNS` in `backupService.ts`.
- The e2e backup test uses `page.request.get()` directly against `http://localhost:3001/export`
  rather than clicking the `<a>` link (cross-origin anchor clicks don't surface as Playwright
  download events). The UI test separately asserts the link is visible.
