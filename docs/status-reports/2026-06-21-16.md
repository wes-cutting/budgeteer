## What #16 built

### A11y pass — challenge-first approach

Ran `@axe-core/playwright` against every user-facing view before writing any fixes. Found 2
serious/critical violation categories:

**Violation 1: `target-size` (WCAG 2.5.8 AA) — fixed via `apps/web/src/index.css`**

Browser-default height for `<button>`, `<input>`, `<select>` is ~19–21 px (< 24 px minimum).
The rule fires when touch-target height AND adjacent-element spacing both fall below 24 px.
The TemplatesView, BudgetVsActualView, and others with dense form layouts failed; the Dashboard
(wider spacing) happened to pass. Fix: `min-height: 24px` on all three element types in a new
global CSS file imported from `main.tsx`.

**Violation 2: `dlitem` (WCAG 1.3.1 A) — fixed by removing `role="status"` from `<dl>`**

Three analysis views (`CreditView`, `PayoffView`, `ForecastView`) used `<dl role="status">`.
Overriding the `<dl>` ARIA role removes its `definition-list` semantics, making `<dt>`/`<dd>`
children structurally invalid per axe. Fix: remove the `role="status"` attribute from each
`<dl>` (the `<dl>` is static display data, not a live region — no live-region behavior was
actually needed).
