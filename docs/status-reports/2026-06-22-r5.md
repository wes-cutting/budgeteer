## Challenge findings (before building)

The kickoff asked to **decide the data path first**. I read the code and confirmed the recommended path:

- **Independent fetch over folding into the core `Promise.all`.** The inline figures are *auxiliary* —
  the Dashboard must still render accounts and envelopes if the budget endpoint is slow or fails. So the
  fetch sits beside the core load (and beside the `R2` badge fetch) in the same `useEffect`, with its own
  `.catch` that leaves `budgetByEnvelope` `null` (→ no inline figures) rather than setting `loadError`.
  This mirrors `R2` exactly.
- **Join by `envelopeId`, render only ACTIVE rows.** The report's rows are keyed into a
  `Map<envelopeId, BudgetVsActualRow>`; the active `EnvelopeList` rows look themselves up by `e.id`. The
  **archived** list is never joined (it renders as before). Even though the budget report *can* include an
  archived envelope (when it has a target or spend), that row is simply never looked up.
- **No faked `$0`.** `EnvelopeBudgetInline` returns `null` unless `row.targetCents !== null`. An envelope
  with no target shows nothing extra — the daily view stays uncluttered, and a `$0` figure would be a lie
  (no budget set ≠ a zero budget). `spent`/`remaining` only appear alongside a real target.
- **Default month = current calendar month.** `currentMonth()` = `new Date().toISOString().slice(0, 7)`,
  matching `BudgetVsActualView`'s `thisMonth()`. (The web `Api.getBudgetVsActual(month)` requires the arg;
  the server defaults to the current month when the query param is omitted, so the two agree.)
- **Accessibility: text, not colour.** The figures are a labelled `<span>` ("Target: … · Spent: … ·
  Remaining: …"); `remaining`'s over-budget case is conveyed by its sign (`-$…`), never by colour alone.
  The existing Dashboard axe scan (`e2e/a11y.spec.ts`) still passes.
- **Freshness without cache wiring.** `App` conditionally renders the Dashboard keyed on `view.name`, so
  it **remounts** (re-fetches) on back-navigation. Setting a target in the Budget view and returning to
  the Dashboard therefore shows the new figure with no extra plumbing — the same mechanism `R2` relies on.

**Decision:** an independent mount-time `getBudgetVsActual(currentMonth())` fetch on the Dashboard, joined
to the active envelope rows by `envelopeId`, rendering `target · spent · remaining` only where a target is
set. `Dashboard.tsx` + its tests + one e2e only. No schema/API/domain change.

## What R5 built

### `Dashboard.tsx` (front-end only)

- New `currentMonth()` helper (`"YYYY-MM"`) for the target join.
- New `budgetByEnvelope` state (`Map<string, BudgetVsActualRow> | null`), populated by an **independent**
  `api.getBudgetVsActual(currentMonth())` fetch in the existing mount `useEffect` (beside the core load and
  the `R2` badge fetch), with a silent `.catch` that leaves it `null`.
- New `EnvelopeBudgetInline` component: renders the labelled `target · spent · remaining` span for one
  envelope row, **only when `targetCents !== null`** (else `null`); `remaining` shows `—` if ever null and
  otherwise `formatCents(remainingCents)` (sign preserved).
- `EnvelopeList` gains an optional `budgetByEnvelope` prop and looks up `budgetByEnvelope?.get(e.id)` for
  each **active** row, passing it to `EnvelopeBudgetInline`. The archived list and all existing row
  controls are unchanged.

No other source file changed. **No `api.ts`/`fakeApi.ts` change** — the endpoint (`getBudgetVsActual`) and
its fake already existed (from `#12`).

## Commands & gotchas (cold-start)

Gate: `npm run typecheck && npm run lint && npm test && npm run test:e2e && npm run format && npm run build --workspace @budgeteer/web && npm audit --omit=dev --audit-level=critical`

- **SCA note:** `npm audit` reports 3 **high**-severity advisories (kysely chain); the gate is
  `--audit-level=critical`, so it passes (exit 0). Pre-existing, not introduced by R5.
- **The inline figures come from a *separate* async fetch** than the envelope list. In tests, `await` the
  figures (`findByText(/Target: …/)`) rather than asserting synchronously after the list renders.
- **Only ACTIVE envelope rows show figures, and only when a target is set.** The archived list is never
  joined; an envelope with no target renders exactly as before (no faked `$0`).
- **Freshness is mount-driven:** `App` remounts the Dashboard on back-navigation, so a target set in the
  Budget view appears on return with no cache wiring.
- `reuseExistingServer: !process.env.CI` — kill stale `:3001`/`:5173` before e2e if you ran the preview
  (R5 changed no server code, so a reused dev server is fine, but a clean start is safest).
- **testing-library `getByText` matches an element's own direct text nodes** — `EnvelopeBudgetInline`'s
  span holds all three figures as text, so `getByText(/Target: \$200\.00/)` resolves it uniquely.
