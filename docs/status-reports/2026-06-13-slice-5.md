control opens the `AllocationEditor` pre-filled with the transaction's current allocations;
saving replaces them via the existing `PUT /transactions/:id/allocations`, preserving the
sum invariant. Works on fully-allocated, partial, and opening transactions; reducing a split
returns the leftover to **needs-allocation**. A thin UI slice — **no new API**. **Next:
archive-an-envelope (#6), then the analysis area.**
