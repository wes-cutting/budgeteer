  whole gate. Also ticked the spike report's own "discard once absorbed" follow-up.
- **Recorded the two accepted notes** (no code-behavior change — documentation only, per the review's
  "leave documented as accepted, no action"):
  - **(a) `bigint`→`Number()` precision.** Postgres stores amounts as `BIGINT`; the API narrows them
    to a JS `number` (`Number(row.amount_cents)`), which is **exact only to `2^53−1` cents ≈ $90T** —
    far beyond any V1 balance. Home: the `Cents` type doc in
    [`packages/domain/src/money.ts`](../../packages/domain/src/money.ts) + a conventions note in
    [`05_DATA_MODEL`](../05_DATA_MODEL.md).
  - **(b) `normalizeName` vs the DB key.** The unique index is `lower(btrim(name))`; the domain's
    `normalizeName` **also** collapses internal whitespace — a strict **superset** of the index's
    equivalence. They agree in practice because the service persists the *normalized* name (the value
    the index sees is already collapsed). Home:
    [`packages/domain/src/naming.ts`](../../packages/domain/src/naming.ts) + the same
    [`05_DATA_MODEL`](../05_DATA_MODEL.md) note (detail already in
    [`eh3`](2026-06-15-eh3.md)).
