**Resume here:** The **fourth and last UX12 thread** is done, so **UX12 is complete** (all four polish
threads — skeletons · destructive confirms · inline validation · toasts — shipped). This slice ships
**thread 4 — success toasts:** every successful mutation was **silent** (the global quick-add even
navigates away, confirming nothing), and now fires a **transient, auto-dismissing toast**. A new
**`ToastProvider` / `useToast()`** (`apps/web/src/ui/Toast.tsx`) on **`@radix-ui/react-toast`** — the
last of the "hard interactive a11y widgets" ADR-0005 reserved for Radix, and the **only UX12 thread

toast **survives** the quick-add's navigate-away). It's wired to the successful-mutation set: create
**account/envelope/transaction** (register + quick-add) · **transfer** · **move money** · **archive**
account/envelope · **delete** template. A11y: Radix mirrors each toast into a **`role="status"` +
`aria-live="polite"`** live region (`type="background"`, **never** `role="alert"`); entrance is
**transform-only** (no text-opacity fade — the UX3 route-fade trap); reduced-motion-safe; explicit
**Dismiss** button; state carried by **text**, not the decorative `--color-success` accent. Proven
**axe-clean light AND dark with the toast visible** before any wiring, so **no ADR** (ADR-0005 covers
it). Gate green: **354 Vitest passing** (+5; 1 pre-existing unrelated carry — §4) **+ 89 e2e** (+3),

## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Why this thread last / ADR-or-not** | Success toasts were saved for last deliberately — the only UX12 thread needing a **new dependency**. **No ADR:** ADR-0005 already reserves Radix for the hard a11y widgets and names **toast**; the a11y risk (an auto-dismissing, animated, live-region toast) was retired **before wiring** by proving it axe-clean light+dark with the toast visible. |
| **A11y mechanism** | The **visible** toast is a plain `<li>`; Radix mirrors its content into a **separate** `role="status"` + `aria-live="polite"` live region (we pass `type="background"` — never `foreground`/assertive/`role="alert"`). This is Radix's announce model and satisfies "polite status, not alert". |
| **No-provider no-op** | `useToast()` returns a silent no-op when no `ToastProvider` is above it — a toast is an *auxiliary* success affordance (the needs-badge degrade pattern), so the many views that unit-test by direct render need no wrapper. Tests that assert a toast wrap the view in `<ToastProvider>`. |
| **Bundle headroom shrank** | +4.63 KB gz → **116.31 KB**, ~3.7 KB headroom (was ~8.3). This is the biggest single add since the charts. **`UX13`–`UX15` must measure against the tighter budget** — a second new dependency would likely breach 120 KB. |
| **jsdom pointer-capture polyfill** | Radix Toast's swipe handler calls `hasPointerCapture`, absent in jsdom v25 → stubbed in the web test setup (real Chromium has it). A one-time env shim, not a product change. |
| **E2E selector scoping (same-change consequence)** | Introducing an app-wide `role="status"` live region made 3 pre-existing **bare** `getByRole("status")` assertions ambiguous (reconcile ×2, recurring post-due) → scoped to their notice with `.filter({ hasText })`. These passed on `main` and were broken **by this slice**, so fixing them is in-scope (distinct from the two unrelated carries below). |
| **Test-env: toasts don't auto-dismiss headless** | In headless Playwright the window is unfocused, so Radix **pauses** each toast's 5s timer — toasts linger for the run. Real (focused) usage dismisses them. Selectors are scoped so this is inert; not a product bug. |
| **Pre-existing carry (recurring, unrelated) — RE-CONFIRMED on clean `main`** | `apps/api/test/recurring.test.ts` › *"post-due generates every due occurrence…"* fails (expects 4 posted, gets 1). A relative-date fixture vs. the sandbox clock — **not** this slice's regression (this slice is web-only). Not touched. |
| **Pre-existing carry (forecast e2e, flaky, unrelated)** | The cash-flow forecast e2e's `getByLabel("Account")` strict-mode collision (documented UX10/UX11 §4). **Did not trip** this run (89/89). Not touched. |

- **Success toast** = the `ToastProvider`/`useToast()` primitive (`ui/Toast.tsx`). To confirm a
  mutation: call `const { showToast } = useToast();` in the container that owns the success callback,
  then `showToast("Account created")` on success. The provider is already mounted at the app root — a
  new call site just needs `useToast()`. **Don't** call it in a leaf form's isolated unit test without
  a `ToastProvider` wrapper (it silently no-ops there — by design).
- **Announce, not alert:** a toast is `role="status"`/polite. For a whole-form *error*, keep using
  `Alert` (`role="alert"`); for a *field* error, `FieldError`. Toasts are for **success**.
- **Bundle:** 120 KB gz ceiling, 116.31 KB now (~3.7 KB headroom — the Radix Toast was the last big
  add). Measure any new dependency against this tighter margin.
- **Known carries (not this slice's regressions):** `recurring.test.ts`'s post-due occurrence-count
  case, and a flaky `getByLabel("Account")` collision in the forecast e2e — both reproduce on `main`,
  see §4.
