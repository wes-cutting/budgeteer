## Bug found & fixed (via the HTTP smoke)

A bodyless request (DELETE) carrying `content-type: application/json` tripped Fastify's
empty-body check, and the error handler **masked the 4xx as a 500**. Fixed three ways:
(a) the web client omits `content-type` when there's no body; (b) the server tolerates an
empty `application/json` body; (c) the error handler **preserves the original status** (only
real 5xx are logged + generalized). Locked with a regression test. *(Integration tests via
`inject` had missed it because `inject` doesn't set that header — the HTTP smoke is what caught it.)*
