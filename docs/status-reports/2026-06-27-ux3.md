## 6. Commands & gotchas (cold-start)

```sh
npm install
npm run typecheck && npm run lint && npm run format
npm test                 # 283 Vitest (node + jsdom)
npm run test:e2e         # 52 Playwright (incl. e2e/a11y.spec.ts axe light+dark, e2e/routing.spec.ts)
npm run build --workspace @budgeteer/web   # 91.4 KB gz JS, 2.2 KB gz CSS
```
- Routing lives in `apps/web/src/`: `routes.tsx` (route map + adapters) · `AppShell.tsx` (shell) ·
  `api-context.tsx` (`useApi()`). View components still take `api` as a **prop** (their unit tests
  render them directly); route adapters inject it from context.
- **Navigate via the shell in e2e:** use the `e2e/setup.ts` helpers (`goToDashboard`,
  `openAnalysis`, `openNeeds`, `openTemplates`, `openRecurring`) — they click the shell `NavLink`s.
  There are no more "← Dashboard" buttons.
- **a11y gotcha (learned here):** never animate **opacity** on a wrapper of text — a mid-flight
  opacity < 1 blends descendant text toward the background and trips the axe contrast gate on muted
  text. Use a **transform** for route transitions.
- Account/envelope **deep links** need the SPA history fallback: Vite dev (`npm run dev`) and `vite
  preview` both serve `index.html` for unknown paths, so refresh on `/accounts/:id` works.
