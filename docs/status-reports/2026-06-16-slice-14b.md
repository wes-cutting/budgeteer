## What #14b did

**Challenged the plan, settled the modeling with the owner.** The owner had already chosen (after
`#14a`) to build `#14b` now rather than defer it. The two `#14b` modeling forks were then put to the
owner (FEAT-014b §11):

- **How to model a loan → add a `kind='loan'` account type.** Payoff applies to installment debt, but
  the schema had no loan kind (only checking/savings/credit/cash/other). Options were (A) add
  `kind='loan'`, (B) a kind-agnostic original-principal store, or (C) reuse `kind='credit'`. The owner
  chose **(A)** — the **symmetric** model: credit→limit→utilization, loan→original→payoff, with the
  original-principal store restricted to `kind='loan'` exactly as `credit_limits` is restricted to
  `kind='credit'`. A **reversible, additive** enum extension (like envelope `sinking_fund`) — a
  **definition**, not an ADR.
- **Original principal over time → single (not effective-dated).** The owner chose one stored original
  per loan, applied throughout the trend (matching `#14a`'s non-effective-dated limit) — a **definition,
  no ADR**. Effective-dated / refinance-aware originals can layer on later.

**No spike — said so explicitly.** `#13` was the analysis area's one genuine unknown (a forward
projection). `#14b` **aggregates** an existing derived balance + one stored number. The only "new" logic
is the payoff formula — a one-line `1 − owed/original` over the **same cumulative monthly trend `#14a`
already proved**. (A true forward projection — an amortization schedule / payoff-date estimate — would
be spike-worthy, and is explicitly out of scope.)

**The build (vertical):**

- **Domain (pure)** — new [`loan.ts`](../../packages/domain/src/loan.ts): `debtPayoff` (per-loan
  owed/paid-down/payoff + cumulative monthly **trend** + portfolio **roll-up**) over `payoffBps`
  (`round((1 − owed/original) × 10000)` — **truthful, not clamped**). Reuses `#14a`'s `CreditFlow`
  per-period shape. No I/O. Exported from the domain index. `ACCOUNT_KINDS` gains `'loan'`.
- **API (read + a tiny store + the new kind)** — new
  [`loanPrincipalService`](../../apps/api/src/services/loanPrincipalService.ts) (`set`/`clear`,
  restricted to `kind='loan'`) mirroring `creditLimitService`; new
  [`analysisService.debtPayoff`](../../apps/api/src/services/analysisService.ts) (structurally identical
  to `creditUtilization`); routes `GET /analysis/debt-payoff`, `PUT`/`DELETE
  /accounts/:id/original-principal`. The accounts `kind` check is **evolved** idempotently to allow
  `'loan'` (drop inline + add named, like the FEAT-007 transactions-kind evolution). **One new table**
  (`loan_principals`); **no balance-view change**.
- **UI (thin)** — new [`PayoffView`](../../apps/web/src/PayoffView.tsx): a roll-up `<dl role="status">`,
  a per-loan `<table>` with an inline **original-principal** editor, and per-loan **trend** tables.
  Reached from a Dashboard **Payoff** button. Every ratio is **text** (percentage, "overpaid", "paid
  off", `— (set an original principal)`); the Add-account form now offers the **loan** kind.

## Commands & gotchas (cold-start)

Gate is unchanged: `npm run typecheck`, `npm run lint`, `npm test`, `npm run test:e2e`, `npm run format`, `npm run build --workspace @budgeteer/web`.

- `npm run format` is `prettier --check .` — checks source `.ts`/`.tsx` but **ignores `docs/**` and
  `*.md`**, so run `npx prettier --write .` before the gate (new **source** files trip it; new docs don't).
- `npm run lint` is a **zero-warning** gate; unused test bindings fail it (assign only what you use).
- `npm run test:e2e` needs Chromium once (`npx playwright install chromium`); it boots both servers
  (web on `:5173` to match the API CORS allowlist) and the journey now ends with a **Payoff** step.
- The `.claude/launch.json` `api` server is `npm run start` (**no watch**) — restart it to pick up API
  changes (incl. the new `kind='loan'` migration) during local preview.
- **The new kind** is added in **three** places (kept in sync): the DB check (`accounts_kind_chk`), the
  domain `ACCOUNT_KINDS`, and the web `AccountKind`/Dashboard list. Any future kind change must touch all
  three. The accounts-kind-check evolution is idempotent (drop inline + add named), like the FEAT-007
  transactions-kind evolution.
- **Owed sign:** a loan in debt has a **negative** derived balance; `owed = −balance`. Open a loan with
  its principal as a **negative** starting balance (e.g. `"-10000.00"`). The trend's final cumulative
  **must** equal the current owed — an asserted invariant.
- CI (`.github/workflows/gate.yml`) stays **manual-only** (`workflow_dispatch`) — unchanged.
