## Challenge findings (before building)

The kickoff flagged three questions. Resolved by reading `transferService.ts`,
`envelopeService.ts`, and `Dashboard.tsx` first:

- **Transfer legs on an archived account?** The transfer **create** path already rejects
  archived accounts (`400`, existing guard — see `06_API_CONTRACT` §transfers, "archived
  account"). That is the only money-moving path that needed a guard, and it was already
  there. **No new block written.**
- **Can you still view existing transactions on an archived account?** **Yes — by design.**
  Archived accounts remain in the `GET /accounts` payload (flagged via `archivedAt`) and
  render in the Dashboard's "Archived accounts" section with their open button intact, so
  the register (and its transaction list) stays reachable. This mirrors envelope archive:
  history is preserved, not hidden.
- **Delete?** Also still works — delete buttons live in `AccountRegister`, which is reachable
  for archived accounts. Archiving is reversible bookkeeping, not a lock.

**Decision:** a pure mirror of `#6`. No domain change, no new guard, no schema change.

## What R7 built

### `accountService.setArchived(id, archived)` (service layer)

Household-scoped existence check → `404` if missing → set `archived_at` to `new Date()` or
`null` → return the refreshed `AccountView`. Identical shape to `envelopeService.setArchived`.

### Two new routes in `server.ts`

- `POST /accounts/:id/archive` — `200 { account }`, `404` if missing
- `POST /accounts/:id/unarchive` — `200 { account }`, `404` if missing

(`POST`, mirroring the envelope endpoints — the roadmap's earlier `PATCH` note was
superseded to stay consistent with `#6`.)

### Frontend

**`apps/web/src/api.ts`:** added `archiveAccount(id)` / `unarchiveAccount(id)` to the `Api`
interface and `httpApi`.

**`apps/web/src/test/fakeApi.ts`:** implemented both (set / clear `archivedAt`).

**`apps/web/src/Dashboard.tsx`:**
- `archiveAccount` / `unarchiveAccount` handlers (refresh accounts on success)
- `AccountList` now splits `active` vs. `archived`; active rows render an **Archive** button
  (`aria-label="Archive {name}"`), archived rows render **Unarchive**
  (`aria-label="Unarchive {name}"`)
- A **"Show archived" / "Hide archived"** toggle reveals an "Archived accounts" section
  (only shown when at least one archived account exists)
