## 4. Notes / decisions

| Item | Note |
| ---- | ---- |
| **Zero drift, zero violations — but only by discipline** | Swapping ~25 hand-copies for the real definitions surfaced no mismatch, and the new lint found no existing boundary breach. That's the manual policing holding — the point of EH12/EH13 is that it no longer has to |

## 6. Commands & gotchas (cold-start)

```sh
npm run typecheck && npm run lint && npm run format
npm test                 # 389 passing
npm run test:e2e         # 94 Playwright — STOP any dev servers on :3001/:5173 first (K20)
npm run build --workspace @budgeteer/web   # 117.32 KB gz
```

- **Web imports server types from `@budgeteer/api/contract` — `import type` ONLY** (the web-zone
  lint fails a runtime import). New view types: define them in the service as usual, re-export
  from `apps/api/src/contract.ts`, and web's `export type *` re-export picks them up.
- **`contract.ts` must never export a value** — it's erased from the bundle only while types-only.
- **The boundary lint speaks for ARCHITECTURE §1**: domain = no framework/datastore/`node:*`;
  web = no datastore/server framework; routes = no direct `db/*`. If a legitimate need trips it,
  that's an architecture conversation, not an eslint-disable.
- **e2e needs to own the stack** (K20): stop dev servers on `:3001`/`:5173` before
  `npm run test:e2e`, restart them after.
- Schema changes are new frozen migration files (`0003-…` + registry + `05_DATA_MODEL`, same
  change); client calendar dates via `apps/web/src/dates.ts` (never `toISOString()`); the server
  requires every user-facing date/month param (EH8).
